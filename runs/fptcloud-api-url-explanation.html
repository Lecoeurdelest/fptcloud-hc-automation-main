Explanation of the FPT Cloud API URL

API URL used: https://console-api.fptcloud.com/api
Project variable: FPTCLOUD_API_URL
Terraform provider argument: api_endpoint

What This API URL Means

The API URL is the base HTTPS endpoint for the FPT Cloud control plane. In this project, Terraform and worker processes use it as the network address for requests such as reading tenant information, reading VPC information, and provisioning cloud resources.

For example, during the connection test Terraform attempted to read tenant data through a request shaped like: https://console-api.fptcloud.com/api/v2/tenant/FCI-L1-ORG. That shows that https://console-api.fptcloud.com/api is treated as the base URL, while paths such as /v2/tenant/... are added by the FPT Cloud Terraform provider.

Where It Comes From

Source Reference Meaning
Project infrastructure spec specs/02-INFRASTRUCTURE.md Lists FPTCLOUD_API_URL=https://console-api.fptcloud.com/api as a required runtime environment variable.
Project requirements specs/01-REQUIREMENTS.md Requires provider authentication/configuration to come from environment variables such as FPTCLOUD_API_URL, FPTCLOUD_REGION, FPTCLOUD_TENANT_NAME, and FPTCLOUD_TOKEN.
Connection-check Terraform runs/fptcloud-connect-check/main.tf Passes the URL into the FPT Cloud Terraform provider through the provider argument api_endpoint.
Application configuration code src/hc/config/settings.py Reads FPTCLOUD_API_URL and forwards it to Terraform subprocesses so workers can talk to the FPT Cloud provider.

Purpose in This Project

This repository automates FPT Cloud tenant health checks. Its workers use Terraform modules to create, inspect, and validate resources such as subnets, virtual machines, disks, floating IPs, security groups, and object storage buckets.

The API URL tells the Terraform provider which FPT Cloud control-plane endpoint to contact. Without it, the provider may use a default endpoint or fail if no endpoint is configured. With it, the project can explicitly target the intended FPT Cloud environment.

Why This URL Was Chosen

  • It is the endpoint selected for this project and documented in the repository's infrastructure specification as the required value for FPTCLOUD_API_URL.
  • It matches the variable model used by the project: provider settings are supplied at runtime rather than hard-coded with credentials.
  • It is the most direct candidate for the FPT Cloud public control-plane API because the project documentation explicitly uses it as the example and default endpoint.
  • It lets Terraform construct provider API calls while keeping tenant, region, and token values separate.

Recommendation

Confirm the correct FPT Cloud API endpoint with the platform documentation or tenant administrator. If the tenant requires a regional, private, or console-specific endpoint, update FPTCLOUD_API_URL in the local environment and api_endpoint in the connection-check variables, then rerun terraform plan.
