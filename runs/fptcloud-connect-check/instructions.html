FPT Cloud Connection Check

This check uses Terraform to read the configured FPT Cloud VPC and preview a test subnet. It uses terraform plan, so it does not create, update, or delete cloud resources.

Files Used

File Purpose
.env Provides FPTCLOUD_API_URL, region, tenant name, and token.
runs/fptcloud-connect-check/main.tf Terraform configuration used for the connection check.
runs/fptcloud-connect-check/connect.auto.tfvars Local values for VPC name, VPC ID, subnet name, CIDR, and gateway IP.
scripts/check-fptcloud-connect.ps1 PowerShell helper that loads .env and runs Terraform.

Expected API URL

The project should use: https://console-api.fptcloud.com/api

Run the Check

Open PowerShell from the repository root and run:

.\scripts\check-fptcloud-connect.ps1

If Terraform is installed but not available on PATH, pass the full executable path: 

.\scripts\check-fptcloud-connect.ps1 -TerraformPath "$env:LOCALAPPDATA\Microsoft\WinGet\Packages\Hashicorp.Terraform_Microsoft.Winget.Source_8wekyb3d8bbwe\terraform.exe"

Successful Result

A successful connection shows Terraform reading the VPC:

data.fptcloud_vpc.selected: Read complete
selected_vpc_id     = "c9c1cfd6-7926-4a8a-96c6-cb4cd9d4aa19"
selected_vpc_status = "ENABLED"

What the Plan Means

Terraform may show Plan: 1 to add, 0 to change, 0 to destroy. That is only a preview for the test subnet. It confirms Terraform can use the provider and resolve the configured VPC.

Do not run terraform apply unless you intentionally want to create the test subnet.

Troubleshooting

Error Likely Cause
terraform is not recognized Terraform is not installed or the current shell has not refreshed PATH.
lookup ... no such host Wrong API URL, DNS issue, VPN issue, or blocked outbound network access.
401 or credential error The token in .env is missing, expired, or not authorized for the tenant.
VPC not found Check vpc_name, vpc_id, tenant, and region in the configuration files.
